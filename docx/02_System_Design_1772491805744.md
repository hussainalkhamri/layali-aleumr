# وثيقة تصميم النظام (System Design) - نظام "ليالي العُمر"

## 1. الهيكلية العامة (High-Level Architecture)
يعتمد النظام على بنية تطبيق ويب سحابي متجاوب (Cloud-Native Responsive Web App) بأسلوب (Monolith) متطور، تتكون من الطبقات التالية:

1.  **قاعدة البيانات (Database Layer)**:
    *   **MySQL**: لضمان سلامة العمليات الجوهرية (Transactions, ACID)، خاصةً فيما يتعلق بالفواتير والسندات والماليات.

2.  **إطار العمل الرئيسي (Core Framework)**:
    *   يتم تطوير الواجهة الأمامية والخلفية بالكامل باستخدام **Laravel 12**.
    *   استخدام **Blade Templates** لتصيير الواجهات (Server-Side Rendering) لضمان الأمان والسرعة.
    *   لتحويل النظام إلى تطبيق متجاوب وتفاعلي (SPA-like) دون الحاجة لـ React/Vue، سيتم دمج **Livewire 3** أو **Inertia.js (إذا لزم لاحقاً، ولكن Blade + Alpine.js هو الأساس)** حسب التعقيد التفاعلي في شاشات التقويم واختيار الفساتين.
    *   تطبيق مصادقة (Session/Cookie Based) آمن للمقاييس الحساسة.
    *   طبقة الصلاحيات (RBAC/OBAC Middleware) لضمان اقتصار وصول المستخدمين لبيانات فروعهم وأدوارهم المحددة.

3.  **التكامل الذكي (AI Integration)**:
    *   **محرك Gemini**: كلاس (Service) يتم استدعاؤه بنظام مهام مجدولة (Cron Jobs / Laravel Scheduler) في الواجهة الخلفية حيث يجمع البيانات المطلوبة (إحصائيات غير حساسة) ويرسلها لخدمة (Gemini API) لتحليلها واستقبال الردود بـ `JSON` وحفظها للعرض في لوحة التحكم الخاصة بالإدارة.

4.  **تطبيق الواجهة (Responsive Blade)**:
    *   استخدام تصميم **TailwindCSS** متجاوب 100% داخل قوالب Blade.
    *   **بوابة العملاء (Public Website)**: صفحات للزوار المفتوحين تعرض أحدث الفساتين المتاحة، لتحسين الظهور (SEO) وجذب العملاء لمقرات "ليالي العُمر". لا يتم الحجز عبرها بل تُستخدم ككتالوج عرض وتوجيه أرقام التواصل.
    *   **تطبيق الموظف (PWA)**: الموظف يفتح الرابط من جواله (تفتح شاشات مخصصة للهاتف بفضل Tailwind متضمنة واجهة البيع).
    *   **لوحة الإدارة**: المدير يفتح نفس الرابط من الكمبيوتر (تظهر لوحة تحكم كاملة).

## 2. إدارة الصلاحيات والأدوار (RBAC / OBAC)
*   **Role-Based Access Control (RBAC)**: الإدارة المركزية تستطيع إنشاء "أدوار" (موظف، مدير فرع، محاسب، إدارة). سيتم استخدام حزمة مثل `Spatie Permission`.
*   **Object-Based Access Control (OBAC)**: ربط الصلاحية بالنطاق. "الموظف محمد" له دور (المبيعات) ولكن نطاقه مقيد بكيان `branch_id = 1`. سيتم تطبيق `Global Scopes` في خوادم Laravel لمنع وصول الموظف لبيانات فروع لا يتبعها.

## 3. إدارة حالة الفساتين والحجوزات (State Machine & Concurrency)
*   **Conflict Prevention Module**:
    يستند النظام على حساب الفترات لحجب الفستان عن شاشات البحث، مثال:
    المستخدم يبحث عن توافر فستان X. إن تقاطع التاريخ المطلوب بأي حجز نشط (يشمل فترة التنظيف)، يرفض النظام العقد تماماً.

*   **Concurrency Control**:
    استخدام قفل قاعدة البيانات التشاؤمي (Pessimistic Locking `sharedLock` / `lockForUpdate` في Eloquent ORM) عند حفظ الفاتورة لمنع الحجز المزدوج في نفس الثانية.

## 4. طبقة الحماية والتتبع (Security & Audit Trail)
*   **إستراتيجية الإلغاء الوهمي (Soft Deletion)**: استخدام الـ `SoftDeletes` المدمجة بـ Laravel.
*   **سجل العمليات الدقيق (Audit Logs)**: استخدام حزمة `OwenIt/laravel-auditing` لتسجيل أي تعديل في النماذج (Models) وحفظ البيانات القديمة والجديدة.

## 5. التكامل المعماري (AI Integration Flow - Gemini API)
1. في نهاية كل أسبوع، يعمل `Laravel Scheduler`.
2. تُجمّع البيانات غير الشخصية.
3. ترسل مع (System Prompt) موجه إلى **Gemini API**.
4. يحلل (Gemini API) هذا السياق ويرسل مقترحات.
5. يستقبل النظام المقترحات ويحفظها كإشعارات للإدارة.
