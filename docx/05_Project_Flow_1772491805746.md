# تدفق العمل ودورة الحياة (Project Flow) - نظام "ليالي العُمر"

يُقسم تدفق العمل إلى عدة مسارات عمل رئيسية تتفاعل مع بعضها.

## 1. دورة حياة حجز فستان (Booking Flow)
1. **البداية**: عميلة تدخل الفرع وتطلب فستان لتاريخ زفاف (15 نوفمبر).
2. **البحث المفلتر**: الموظف يفتح تطبيق الويب (من الآيباد أو الجوال) ويدخل "15 نوفمبر" في خانة البحث. يفحص النظام (Laravel) التواريخ ويعرض *فقط* الفساتين المتاحة.
3. **الاختيار**: يتم اختيار الفستان وتحديد التوابع وأي تعديلات مطلوبة.
4. **الفاتورة (العقد)**: يُدخل الموظف (المبلغ المتفق عليه للتأجير/البيع)، ويستلم (المبلغ المقدم نقداً).
5. **التسجيل (Pessimistic Locking)**: يضغط "تأكيد". يقوم `Laravel` بقفل سطر الفستان في قاعدة البيانات لأجزاء من الثانية لحفظ العقد، ينشئ سجل `BOOKING_INVOICE` وينشئ `RECEIPT` للمبلغ المقبوض، وتتغير حالة الفستان ضمنياً للفترة المحجوزة.
6. **سجل التدقيق**: نظام التدقيق الآلي يخزن الحدث سراً في `AUDIT_LOG` لتوثيق تفاصيل العملية وتوقيتها والفروقات.

## 2. دورة حياة يوم تسليم الفستان (Output Event)
1. **الصباح (التنبيهات)**: يظهر إشعار في لوحة تحكم `Blade` الخاصة بـ (مدير الفرع والموظفين): "اليوم يوجد 5 فساتين للخروج للعملاء...".
2. **قدوم العميلة للاستلام**: تبحث الموظفة عن العقد (الفاتورة).
3. **الدفع المتبقي**: النظام يُظهر بوضوح أن هنالك مبلغ متبقي (مثلاً 10,000 ريال). يمنع النظام تغيير حالة الفاتورة إلا بإنشاء سند قبض للمبلغ المتبقي (`RECEIPT` جديد).
4. **التسجيل**: تُدفع القيمة، يُطبع سند استلام، وتتغير حالة الحجز إلى (Active - تم خروج الفستان).

## 3. دورة حياة عودة الفستان والتنظيف (Return & Maintenance)
1. **الاستلام**: تعود العميلة لتسليم التأجير. يُفحص الفستان.
2. **التغيير للحالة الداخلية**: حالة الفستان تتغير إلى `Cleaning` (تنظيف).
3. **الانتهاء المرتبط**: بمجرد إنهاء التنظيف، يعود الفستان آليًا أو يدوياً لحالة `Available`.

## 4. تدفق عمل المستشار الذكي (Gemini API Flow)
1. **تجميع البيانات المجدول**: مجدول المهام الخاص بـ Laravel (`cron job`) يستيقظ دورياً (مثلا نهاية الأسبوع).
2. **تحضير البيانات (Sanitization)**: يقوم `GeminiAiClient` بسحب الإحصائيات (أكثر الفساتين حجزا، مبالغ الإيرادات لكل فرع، أعداد مرات الإيجار لكل فستان). يتم إرسال الأرقام الهيكلية للموديلات دون أي بيانات شخصية تخص العملاء.
3. **تقديم الاقتراح الاستراتيجي (API Call)**: ترسل الحزمة كـ Prompt لـ `Gemini API`.
4. **تخزين الاقتراح**: يرد Gemini بالاقتراح والنصائح في هيئة JSON أو Text، فيُحفظ في جدول الـ `SYSTEM_INSIGHT`.
5. **دور الإنسان (المدير)**: المدير يفتح لوحة التحكم الخاصة به (Admin Dashboard)، يجد بطاقات الإشعارات من المساعد الذكي، فيقوم بقراءتها لاتخاذ القرارات اللازمة يدوياً.

## 5. تدفق طلبات النقل الداخلي (Transfer Flow)
1. طلب النقل يُنشأ من فرع، وتتلقى الإدارة أو الفروع إشعاراً في لوحة التحكم.
2. الفستان المراد نقله تتغير حالته إلى `In Transit` بمجرد شحنه.
3. يستلم الفرع المستفيد الفستان ويضغط في التطبيق "تم الاستلام".
4. حقل `current_branch_id` للفستان يتحدث للفرع الجديد عبر الصلاحيات (OBAC) ليتم إدراجه في قوائمهم.
